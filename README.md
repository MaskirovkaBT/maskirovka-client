# Maskirovka Client

TUI-клиент для просмотра и управления юнитами BattleTech. Построен на фреймворке Textual.

## Описание

**Maskirovka Client** — это текстовый пользовательский интерфейс (TUI) для работы с каталогом боевых машин вселенной BattleTech. Приложение предоставляет удобный интерфейс для просмотра юнитов, управления ангаром, фильтрации и сортировки с клавиатурной навигацией.

## ⚠️ Дисклеймер

**BattleTech** — зарегистрированная торговая марка [Catalyst Game Labs](https://www.catalystgamelabs.com) / Topps.

Все данные юнитов, названия и игровые механики являются собственностью соответствующих правообладателей.

Этот проект является **неофициальной фанатской разработкой**, созданной исключительно в образовательных целях.

Проект не аффилирован с Catalyst Game Labs, не одобрен и не спонсируется ею.

Исходный код приложения распространяется под лицензией MIT (см. LICENSE).

## Функциональность

- 📚 **Просмотр справочников**: списки эр и фракций
- 🔍 **Поиск юнитов** по выбранной эре и фракциям
- 📦 **Управление ангаром**: добавление юнитов, изменение количества, комментарии
- 🗂️ **Группировка в ангаре**: автоматическая группировка по базовому имени (все Atlas отображаются как одна группа)
- 🎨 **Подсветка в поиске**: юниты из ангара выделяются зелёным цветом
- 🔍 **Фильтрация юнитов** по множеству параметров:
  - Тип и название юнита
  - Характеристики (PV, размер, броня, структура и др.)
  - Оружейные показатели (short, medium, long, extreme)
  - Особые способности (specials)
- 📊 **Сортировка** по различным полям (название, PV, роль, характеристики)
- 📄 **Пагинация** результатов
- ⌨️ **Клавиатурная навигация**: полное управление без мыши

## Стек технологий

- **Textual** — современный TUI фреймворк для Python
- **httpx** — асинхронный HTTP клиент с переиспользованием соединений
- **Pydantic + pydantic-settings** — валидация данных и настроек
- **Python 3.9+**

## Установка и запуск

### Требования

- Python 3.9+
- Ссылка на сервер с данными о юнитах
- Установленные зависимости из `requirements.txt`

### Установка зависимостей

```bash
pip install -r requirements.txt
```

### Конфигурация

Создайте файл `.env` в корне проекта (или рядом с .exe):

```
API_BASE_URL=http://127.0.0.1:8000
```

### Запуск приложения

```bash
textual run maskirovka.py
```

## Интерфейс и навигация

### Горячие клавиши

#### Глобальные
| Клавиша | Действие |
|---------|----------|
| `Tab` | Переключение между вкладками |
| `q` | Выход |

#### Вкладка "Поиск"
| Клавиша | Действие |
|---------|----------|
| `Tab` / `Shift+Tab` | Переключение между блоками (эры → фракции → таблица) |
| `↑` / `↓` | Навигация по спискам |
| `Space` | Выбор/снятие выбора фракции |
| `Enter` | Открыть детали юнита |
| `Ctrl+s` | Загрузить юниты по выбранным критериям |
| `Ctrl+o` | Открыть окно сортировки |
| `Ctrl+f` | Открыть окно фильтрации |
| `Ctrl+←` / `Ctrl+→` | Предыдущая / следующая страница |
| `Ctrl+a` | Добавить выбранный юнит в ангар |
| `Escape` | Закрыть модальное окно |

#### Вкладка "Ангар"
| Клавиша | Действие |
|---------|----------|
| `↑` / `↓` | Навигация по таблице |
| `Enter` | Открыть детали юнита / свернуть-развернуть группу |
| `a` | Увеличить количество юнита на 1 |
| `d` | Уменьшить количество юнита на 1 |
| `Escape` | Закрыть модальное окно |

### Фильтрация

Окно фильтрации (`Ctrl+f`) позволяет задать:

| Параметр | Описание |
|----------|----------|
| `title` | Поиск по названию (частичное совпадение) |
| `unit_type` | Тип юнита (Mech, Vehicle и др.) |
| `role` | Роль (Brawler, Scout и др.) |
| `specials` | Особые способности (режимы: OR/AND) |
| Числовые | PV, SZ, Short, Medium, Long, Extreme, OV, Armor, Struc, Threshold, MV |

Режимы сравнения для чисел: `eq` (равно), `gt` (>), `gte` (≥), `lt` (<), `lte` (≤)

### Сортировка

Окно сортировки (`Ctrl+o`) позволяет сортировать по:
- `title` — название
- `pv` — Point Value (стоимость)
- `role` — роль
- `short`, `medium`, `long` — оружейные показатели
- `armor`, `struc` — броня и структура
- `mv` — движение

Порядок: `asc` (по возрастанию) / `desc` (по убыванию)

## Архитектура

### Dependency Injection (DI)
- `HangarService` создаётся один раз в `Maskirovka.__init__()` и передаётся в виджеты/экраны
- `ApiClient` переиспользует HTTP-соединение (не создаётся на каждый запрос)

### Message Passing
Все события между компонентами проходят через Textual Messages:
- `SearchUnitSelected` — выбор юнита в поиске
- `HangarUnitSelected` — выбор юнита в ангаре
- `AddToHangar` — добавление в ангар

### HangarTableAdapter
Адаптер для инкапсуляции работы с таблицей ангара. Обеспечивает:
- Получение информации о юните под курсором
- Оптимистичное обновление количества (UI обновляется мгновенно)
- Управление группами (обновление количества, удаление пустых)
